End-to-end pipeline

1. Ingest

Pull papers from arXiv, OpenAlex, CrossRef or upload PDFs.

2. Parse

PDF → text, section detection, sliding-window chunking.

3. Extract

NER (regex + spaCy + MatSciBERT) + relation extraction.

4. Knowledge Graph

Build NetworkX / Neo4j graph, recommend materials.

5. Vector Index

Embed chunks (sentence-transformers) into ChromaDB for RAG.

6. Predict

XGBoost / RF / GBM regressor for capacity, cycle life, voltage.

7. RAG Chat

Ask questions about the indexed corpus – cited answers.

8. Visualize

Interactive Plotly charts + pyvis graph network.

Quick start
# 1. Install dependencies
pip install -r requirements.txt

# 2. Configure environment
cp .env.example .env
# Edit .env and add your OPENAI_API_KEY

# 3. Initialize database
python manage.py migrate
python manage.py createsuperuser

# 4. Seed sample data
python manage.py seed_data

# 5. Run extraction pipeline
python manage.py extract_all
python manage.py build_kg
python manage.py train_models
python manage.py index_all

# 6. Launch the server
python manage.py runserver
# Visit http://localhost:8000
What makes this research-grade?
  • Multi-source ingestion – arXiv, OpenAlex, CrossRef, Materials Project API, manual PDF upload.
  • Ensemble NER – regex patterns + spaCy + HuggingFace MatSciBERT transformer.
  • Typed relation extraction – HAS_PROPERTY, SYNTHESIZED_BY, MEASURED_BY, DOPED_WITH, etc.
  • RAG with citations – every LLM answer cites the source chunks (no hallucination).
  • 6 ML targets – capacity, cycle life, voltage, energy density, safety, cost.
  • Async pipeline – Celery + Redis for long-running ingestion jobs.
  • Production-ready – Docker, Render blueprint, JWT auth, CORS, WhiteNoise.
